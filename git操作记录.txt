初始化仓库（如未初始化）
git init


配置远程地址
	查看远程：
	git remote -v
	修改远程地址：
	git remote set-url origin https://github.com/chenmoon1111/claude-Code-zh-cn-v1.git
添加文件到暂存区
git add .

提交更改
git commit -m "your commit message"

处理远程已有内容
如果远程已有初始 commit，先拉取合并：
git pull origin main --allow-unrelated-histories
解决冲突后再次提交。

推送到远程
git push -u origin main
跨平台换行符统一

在仓库根目录添加 .gitattributes 文件：
gitattributes
* text=auto eol=lf
*.ps1 text eol=crlf
*.sh text eol=lf
*.md text eol=lf

提交并推送：
git add .gitattributes
git commit -m "Add .gitattributes for cross-platform consistency"
git push origin main


   情况 1：推送到错误的仓库
    1 # 删除错误的远程仓库关联
    2 git remote remove origin
    3
    4 # 添加正确的仓库
    5 git remote add origin https://github.com/正确的地址.git
    6
    7 # 重新推送
    8 git push -u origin main
   ---

   情况 2：想撤回已经推送的提交
    1 # 回退到上一个提交（本地）
    2 git reset --soft HEAD~1
    3
    4 # 强制推送到远程（谨慎使用）
    5 git push -f origin main
   ---

   情况 3：想删除整个远程仓库

   在 GitHub 上：仓库 → Settings → 拉到最下面 → Delete this repository

   然后重新创建。

   分支管理

   查看分支
    1 # 查看本地分支
    2 git branch
    3
    4 # 查看所有分支（含远程）
    5 git branch -a
   ---

   创建新分支
    1 # 创建并切换到新分支
    2 git checkout -b dev
    3
    4 # 或分开写
    5 git branch dev
    6 git checkout dev
   ---

   推送分支到远程
    1 # 推送 dev 分支
    2 git push -u origin dev
   ---

   切换分支
    1 git checkout main    # 切换到 main
    2 git checkout dev     # 切换到 dev
   ---

   合并分支
    1 # 切换到 main
    2 git checkout main
    3
    4 # 合并 dev 分支
    5 git merge dev
   ---

   删除分支
    1 # 删除本地分支
    2 git branch -d dev
    3
    4 # 删除远程分支
    5 git push origin --delete dev
   ---

   常用工作流

   日常开发
     1 # 1. 切换到开发分支
     2 git checkout -b feature/linux-support
     3
     4 # 2. 修改代码...
     5
     6 # 3. 提交
     7 git add .
     8 git commit -m "添加 Linux 支持"
     9
    10 # 4. 推送
    11 git push -u origin feature/linux-support
   合并回主分支
     1 # 1. 切换到 main
     2 git checkout main
     3
     4 # 2. 拉取最新代码
     5 git pull origin main
     6
     7 # 3. 合并开发分支
     8 git merge feature/linux-support
     9
    10 # 4. 推送
    11 git push origin main
   ---

   查看历史
    1 # 查看提交历史
    2 git log --oneline
    3
    4 # 查看最近 5 条
    5 git log --oneline -5
    6
    7 # 查看某个文件的修改历史
    8 git log --follow -p 文件名
